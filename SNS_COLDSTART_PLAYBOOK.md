# アニメログ SNSコールドスタート プレイブック

作成 2026-06-14。目的は **最初の数人の流入と「使う理由」の検証**。Claude×Codex のディベートで収束した戦略。

> **役割分担**: このドキュメントの文面・ターゲット選定・スクリプトは Claude/Codex が用意する。
> **実際の投稿・リプ・DM送信はあなた（アカウント所有者）が手動で行う。** 外部送信は取り消せず、X規約と"中の人"の真正性の観点からも本人実行が必須。

---

## 0. 大前提（なぜこの順番か）

- 勝負どころは「配布」より **「競合(Annict/Filmarks)を使わない理由=wedge」**。それが **「ログイン不要で30秒で"今期/来期の視聴予定"をカード化してXに貼れる」**。
- DNAカード(最推し)のバイラルループは **種ユーザーがいて初めて回る後段**。今は回さない。代わりに **予定カードを"こちらから手渡す"** 形で n=0 でも使う。
- 「数人来た」では何も検証できない。**行動と継続**で測る（下記KPI）。

---

## 1. ファネル目標（KPI）

「流入人数」ではなく以下を週次で追う:

| 段階 | 目標 | 意味 |
|---|---|---|
| 接触 | 20人 | DM/リプ/投稿反応で実際に会話 or 訪問 |
| 起動 | 10人 | 夏アニメを1作以上登録した |
| 継続 | 5人 | 翌週も予定を更新した（リテンションの芽） |
| 拡散 | 3人 | 予定カード/リストをXにシェアした |
| 確信 | 1人 | 「これないと困る」と言った（=PMFの最小シグナル） |

「確信1人」が出たら勝ち筋あり。0人なら wedge を作り直す。

---

## 2. ターゲット選定基準

**狙う人（ヘビー層に絞る。最初は5人）:**
- 毎クール10本以上見ている
- Xで「今期/来期 何見る」「視聴予定」「感想」を**実際に投稿している**
- フォロワー数は問わない（むしろ中小アカの方が反応が濃い）

**狙わない人:**
- アニメをたまに見る程度（継続しない＝検証ノイズ）
- 開発者アカウント（#個人開発で来る層。ユーザーではない。信頼/SEOの種に留める）

**探し方（X検索クエリ例）:**
- `2026夏アニメ 何見る`
- `来期 視聴予定`
- `今期 完走` / `今期 切った`
- `アニメ 見たの忘れ` / `何見てたっけ`

---

## 3. Xローンチ投稿ドラフト（複数案）

> ヒント: 機能ではなく**痛み**から入る。固定ポストにし、カード画像を必ず添付（OGPでなく実画像）。

**案A（痛み起点・本命）**
```
「あのクール、何見てたっけ？」をなくしたくてアニメログ作りました。

ログイン不要で、今期/来期の視聴予定を30秒でカード化してそのままXに貼れます👇
（画像は実際に作った2026春の予定カード）

animelog.jp
#2026年夏アニメ #アニメ好きと繋がりたい
```

**案B（来期予定フック・7月直前用）**
```
2026夏アニメ、何見るか決めた人へ。

視聴予定をカード1枚にして晒せるやつ作りました。ログイン不要・無料。
「見る/見てる/見た」を後から振り返れます。

animelog.jp
#2026年夏アニメ
```

**案C（個人開発コミュニティ向け・別投稿）**
```
個人開発でアニメ視聴記録のPWA作って公開しました。
ログイン不要、クール別管理、視聴予定カードのシェアあたりが推しポイントです。
フィードバックもらえると嬉しいです🙏 animelog.jp
#個人開発
```
※案Cは**ユーザー獲得ではなく信頼/被リンクの種**。期待値を分けて運用。

## 4. リプ/DMテンプレ（"作って渡す"方式）

> 「使って」と言うのではなく、**相手の投稿から予定リストを先に作って手渡す**。これがゼロ初期で唯一効く。スパムにしない。

**リプ（「2026夏何見る」系の投稿に対して）**
```
めっちゃ分かります。よかったら、その作品リストでこういう"視聴予定カード"作れますよ（ログイン不要・30秒）。
試しに作ってみたら貼りますね 👉 animelog.jp
```
→ 反応があったら、相手が挙げてる作品で実際にカードを作って画像を返信する。

**DM（一度やり取りがあった人だけ。コールドDMはしない）**
```
さっきの夏アニメの件、〇〇さんの挙げてた作品でカード作ってみました（画像）。
もしハマりそうなら animelog.jp でそのまま自分用に編集できます。感想もらえたら開発の参考にします！
```

**やってはいけない:**
- 無差別コピペDM
- 初回接触でいきなりリンクだけ投下
- 相手の文脈を読まずに「使って」

## 5. タイミングカレンダー（2026年）

「来期何見る?」は **7/1ではなく6月後半から**始まる。今がもう窓口。

| 時期 | やること |
|---|---|
| 6月後半（今） | 予定カード機能リリース（本PR）→ 友人2〜3人で動作確認＆恥ずかしくないか点検。案A投稿の文面確定。 |
| 6月末〜7月頭 | 夏クール予定が固まる時期。**案A/案Bでローンチ投稿**＋ヘビー層5人に"作って渡す"アプローチ。 |
| 7月前半 | 「1話見た/切った」文脈に寄せて投稿（視聴ステータス機能を見せる）。 |
| 7月中旬 | 「継続管理」文脈。翌週も使ってる人=継続KPIを確認。 |

## 6. 避けるもの（明示）

- **5ch/アニメ板への自己宣伝** … 大抵敵対的。割に合わない。
- **#個人開発でのユーザー獲得期待** … devは集まるがユーザーではない。信頼/SEOの種に限定。
- **Google Play申請** … 今はオーバースペック（個人アカウントはクローズドテスト要件あり=要確認）。Webリーチが先。
- **DNAカードのループ最適化** … 種ユーザー獲得後。今やらない。

## 7. 計測の仕込み（任意・あると良い）

- 既に Vercel Analytics 導入済み。投稿/リプの流入は **UTM付きURL**（例 `animelog.jp/?utm_source=x&utm_campaign=summer2026`）で出所を分けると、どの文面が効いたか分かる。
- 予定カードの「保存・シェア」イベントを `track('season_card_share')` で計測すると拡散KPIが取れる（実装は別タスク、`@vercel/analytics` の `track` を流用）。
